Stripe Billing Verification
Prerequisites
	1	Stripe Account: You need a Stripe account in Test Mode.
	2	Environment Variables: Ensure .env.local has:
	•	STRIPE_SECRET_KEY: sk_test_...
	•	STRIPE_WEBHOOK_SECRET: whsec_... (from Stripe CLI or Dashboard)
	•	NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY: pk_test_... (if needed by client)
	3	Database: Supabase subscriptions table must involve strict RLS or be accessible. Currently logic uses service_role in webhooks, so RLS is bypassed there, but client logic uses standard access.
Manual Setup (Required)
	1	Create Products in Stripe:
	•	Go to Stripe Dashboard > Products.
	•	Create "Pro Plan":
	•	Price: $29 / month.
	•	Copy Price ID (e.g., price_123...).
	•	Create "Team Plan":
	•	Price: $99 / month.
	•	Copy Price ID.
	2	Update Code with Price IDs:
	•	Open  src/components/billing/PricingTable.tsx.
	•	Replace price_PRO_ID_PLACEHOLDER and price_TEAM_ID_PLACEHOLDER with your actual Test Price IDs.
	3	TIPIdeally, put these in Environment Variables like VITE_STRIPE_PRICE_PRO and access via import.meta.env. 
	4	Stripe CLI (for Webhooks):
	•	Run stripe listen --forward-to localhost:8888/.netlify/functions/billing-webhook.
	•	Copy the webhook signing secret to .env.local as STRIPE_WEBHOOK_SECRET.
Verification Steps
1. Upgrade Flow
	1	Log in to Codra.
	2	Navigate to /settings/billing.
	3	Click "Upgrade" on the Pro plan.
	4	You should be redirected to Stripe Checkout.
	5	Enter test card details (e.g., 4242...).
	6	Complete payment.
	7	Verify you are redirected back to Codra.
	8	Check subscriptions table in Supabase. You should see a new record with status: 'active' and plan_id: 'pro'.
2. Usage Tracking
	1	Use the AI features (if connected to  router.ts).
	2	Observe the Usage Meter in Billing Settings increase.
	3	Manually update usage_count in DB to 2000 (limit).
	4	Try to generate again.
	5	Verify "Upgrade Required" error or modal appears.
3. Portal Flow
	1	In Billing Settings, click "Manage Subscription".
	2	Verify redirect to Stripe Billing Portal.
	3	Cancel the subscription.
	4	Verify status updates to active (with cancel_at_period_end: true) in DB (via webhook events).
